# 3个月 AI应用开发执行总览

适用背景：

- 已有 Java 后端开发基础。
- 已完成 Python 基础语法复习。
- 目标是快速转向 AI 应用开发实战。

时间安排：

- 总周期：3 个月。
- 每月 4 周。
- 每周 5 天。
- 每天 2-3 小时。
- 每周约 10-15 小时。

执行原则：

- 用项目带学习，不单独刷概念。
- 每周必须有可运行产出。
- 每个阶段必须完成一个项目。
- 重点投入 AI 增量能力：LLM API、Prompt、RAG、Agent、部署。
- Java 后端已有知识只做迁移，不重复学习。

阶段文件：

- [01_LLM_API与FastAPI_AI_Chat阶段.md](01_LLM_API与FastAPI_AI_Chat阶段.md)
- [02_RAG企业知识库阶段.md](02_RAG企业知识库阶段.md)
- [03_Agent与生产化阶段.md](03_Agent与生产化阶段.md)

总体路线：

```text
第1个月：LLM API + Prompt + FastAPI AI Chat
第2个月：RAG + Embedding + 向量检索 + 知识库问答
第3个月：Agent + Tool Calling + 部署 + 作品整理
```

最终产出：

- AI Chat 后端。
- 企业知识库 RAG 系统。
- AI SQL/日志分析 Agent。
- 一个 Docker 化可部署项目。

每周固定节奏：

```text
Day 1：理解核心概念 + 最小示例
Day 2：实现核心功能
Day 3：推进项目功能
Day 4：异常处理 + 工程化整理
Day 5：测试 + README + 复盘
```

每周验收标准：

- 至少 1 个可运行服务或脚本。
- 至少 3 个手动测试用例。
- 至少 1 份 README 或阶段笔记。
- 能用自己的话讲清楚本周核心链路。

可以快速跳过：

- HTTP 基础。
- RESTful 基础。
- MVC/分层架构思想。
- JSON 基础。
- 数据库基础概念。
- Docker 基础概念。

必须重点练：

- messages 设计。
- Prompt 稳定性。
- structured output。
- streaming。
- Embedding。
- 文档切片。
- 向量检索效果。
- RAG 引用来源。
- Function Calling。
- Agent 工具调用闭环。
