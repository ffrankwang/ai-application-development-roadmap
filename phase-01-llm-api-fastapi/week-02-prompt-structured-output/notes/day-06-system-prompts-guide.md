# Day 6：system prompt 模板练习指南

今天目标：

写出 5 个可复用的 system prompt 模板，并测试它们对回答风格的影响。

## 任务要求

- 新建 prompt 模板文件。
- 至少写 5 个 system prompt。
- 每个 prompt 至少测试 2 个问题。
- 记录 prompt、问题、回答摘要、观察。

## 模板方向

先自己写，不要追求完美：

- Python 学习导师。
- FastAPI 架构顾问。
- SQL 优化助手。
- 日志分析助手。
- JSON 输出助手。

## 一级提示

一个好的 system prompt 通常包含：

```text
角色
任务
输出格式
限制条件
风格要求
```

## 二级提示

你可以使用这个骨架，但内容自己填：

```text
你是...
你的任务是...
回答时请...
不要...
输出格式为...
```

## 自检清单

- 是否有 5 个 prompt？
- 是否每个 prompt 都实际测试过？
- 是否记录了回答差异？
- 是否避免了过长、过泛的 prompt？

## 完成标准

- 5 个 prompt 模板。
- 10 条测试记录。
- 至少 3 条关于 prompt 设计的结论。
