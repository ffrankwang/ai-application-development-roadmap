# Day 7：few-shot 示例练习指南

今天目标：

用 few-shot 让模型学习固定回答风格和输出结构。

## 任务要求

- 选择 1 个任务场景。
- 写 2-3 个输入输出示例。
- 再给一个新输入，让模型按示例风格回答。
- 对比有 few-shot 和无 few-shot 的差异。

## 推荐场景

任选一个：

- SQL 优化建议。
- Python 报错解释。
- FastAPI 接口设计建议。
- 日志错误分析。

## 一级提示

few-shot 的结构：

```text
任务说明
示例1：输入 -> 输出
示例2：输入 -> 输出
新输入
```

## 二级提示

不要只给“好答案”，还要让格式稳定：

```text
问题：
原因：
建议：
示例代码：
```

## 自检清单

- 是否至少有 2 个示例？
- 示例输出格式是否一致？
- 新问题的输出是否跟随格式？
- 是否记录了无 few-shot 对比结果？

## 完成标准

- 至少 1 个 few-shot prompt。
- 至少 5 个测试输入。
- 输出格式稳定度明显提升。
